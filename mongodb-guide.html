❓ Why can't GitHub Pages talk to MongoDB directly?

GitHub Pages only hosts static files — HTML, CSS, and JavaScript. Think of it like a poster on a wall. It can only show what's already printed on it.

MongoDB needs a secret password (your connection string) to connect. If you put that password in your JavaScript file on GitHub, anyone on the internet can read it and steal your database. 🚨

⚠️ Never do this
const uri = "mongodb+srv://admin:mypassword123@cluster0.mongodb.net"

Never put your MongoDB password directly in JavaScript files that go on GitHub. It's like leaving your house key under the doormat.
✅ The safe way
You need a backend server that keeps the password hidden. Your website talks to the backend, and the backend talks to MongoDB. The password never leaves the server.

🔧 The Solution — Render.com (free backend hosting)

Since you're using the free version of GitHub Pages, here's the completely free stack you'll use:

Your Free Tech Stack
  • 🌐 GitHub Pages — hosts your HTML/CSS/JS website (you already have this)
  • ⚙️ Render.com — FREE backend server hosting (Node.js) — this is your "kitchen"
  • 🗄️ MongoDB Atlas — FREE cloud database — you already made an account!

1 Set Up MongoDB Atlas

You already made an account at cloud.mongodb.com. Now let's set it up properly.

1a. Create a free cluster

  1. Log in to cloud.mongodb.com
  2. Click "Build a Database"
  3. Choose FREE / M0 Shared (it's free forever)
  4. Pick any cloud provider (AWS is fine) and a region close to you
  5. Click "Create"

1b. Create a database user

  1. On the left sidebar, click Database Access
  2. Click "Add New Database User"
  3. Choose Password authentication
  4. Username: vcs_admin
  5. Password: click "Autogenerate" and copy it somewhere safe
  6. Under "Database User Privileges" → choose Read and write to any database
  7. Click "Add User"

1c. Allow connections from anywhere

  1. On the left sidebar, click Network Access
  2. Click "Add IP Address"
  3. Click "Allow Access from Anywhere" — this adds 0.0.0.0/0
  4. Click "Confirm"
💡 Why "anywhere"?
Your backend server on Render.com has a changing IP address, so you need to allow all IPs. This is fine because your database still requires a username and password to connect.

1d. Get your connection string

  1. Go to Database (left sidebar) → click "Connect" on your cluster
  2. Choose "Connect your application"
  3. Select Node.js, version 4.1 or later
  4. Copy the connection string — it looks like this:
mongodb+srv://vcs_admin:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority

Replace <password> with the password you saved earlier. Save this whole string — you'll need it in Step 3.

2 Set Up Render.com (Free Backend Host)

  1. Go to render.com and sign up for free (use your GitHub account to log in — easiest)
  2. After signing in, click "New +" → choose "Web Service"
  3. Connect your GitHub account and select your VCS repository
  4. Fill in the settings:
    • Name: vcs-backend
    • Runtime: Node
    • Build Command: npm install
    • Start Command: node server.js
    • Instance Type: Free
  5. Scroll down to "Environment Variables" and add:
    • Key: MONGODB_URI → Value: your connection string from Step 1d
    • Key: ALLOWED_ORIGIN → Value: https://your-username.github.io
  6. Click "Create Web Service" — Render will give you a URL like https://vcs-backend.onrender.com
💡 Why environment variables?
Environment variables are like secret notes the server reads privately. Your GitHub code never sees them — they stay safely on Render's servers.

3 Write the Backend Code

Create these two files in a new folder called backend/ inside your GitHub repository.

backend/package.json

{
  "name": "vcs-backend",
  "version": "1.0.0",
  "main": "server.js",
  "scripts": {
    "start": "node server.js"
  },
  "dependencies": {
    "cors": "^2.8.5",
    "express": "^4.18.2",
    "mongoose": "^8.0.0"
  }
}

backend/server.js

// ── Imports ─────────────────────────────────────────────────
const express  = require('express');
const cors     = require('cors');
const mongoose = require('mongoose');

const app = express();
app.use(express.json());

// ── CORS — only allow your GitHub Pages site ─────────────────
app.use(cors({
  origin: process.env.ALLOWED_ORIGIN  // set this in Render environment variables
}));

// ── Connect to MongoDB Atlas ─────────────────────────────────
mongoose.connect(process.env.MONGODB_URI)
  .then(() => console.log('✅ Connected to MongoDB Atlas'))
  .catch(err => console.error('❌ MongoDB error:', err));

// ── Task Schema (like a table blueprint) ─────────────────────
const taskSchema = new mongoose.Schema({
  title:         String,
  billable:      Number,
  peso_rate:     Number,
  assignTo:      String,
  assignedUsers: [String],
  status:        { type: String, default: 'pending' },
  tag:           { type: String, default: 'coming-soon' },
  uploaded:      String,
}, { timestamps: true });

const Task = mongoose.model('Task', taskSchema);

// ── Routes ───────────────────────────────────────────────────

// GET all tasks
app.get('/api/tasks', async (req, res) => {
  const tasks = await Task.find().sort({ createdAt: -1 });
  res.json(tasks);
});

// GET tasks for a specific user
app.get('/api/tasks/mine/:uid', async (req, res) => {
  const tasks = await Task.find({ assignedUsers: req.params.uid });
  res.json(tasks);
});

// POST create task
app.post('/api/tasks', async (req, res) => {
  const task = await Task.create(req.body);
  res.json(task);
});

// PATCH update task (tag, status, upload)
app.patch('/api/tasks/:id', async (req, res) => {
  const task = await Task.findByIdAndUpdate(req.params.id, req.body, { new: true });
  res.json(task);
});

// DELETE task
app.delete('/api/tasks/:id', async (req, res) => {
  await Task.findByIdAndDelete(req.params.id);
  res.json({ ok: true });
});

// ── Start server ─────────────────────────────────────────────
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => console.log(`🚀 Server running on port ${PORT}`));

4 Update Your Frontend (api-client.js)

In your existing js/api-client.js file, change the API base URL to point at your Render backend:

// Change this one line in api-client.js
const API_BASE = 'https://vcs-backend.onrender.com/api';
//                ↑ replace with your actual Render URL

// Then fetch tasks like this from anywhere in your JS:
const tasks = await fetch(`${API_BASE}/tasks`).then(r => r.json());

// Create a task:
await fetch(`${API_BASE}/tasks`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ title: 'New Task', billable: 60, peso_rate: 25 })
});

5 Deploy and Test

  1. Push the backend/ folder to your GitHub repo
  2. Render will automatically detect the changes and redeploy
  3. Visit your Render URL: https://vcs-backend.onrender.com/api/tasks — you should see [] (empty array)
  4. Open your GitHub Pages site — it will now read/write from MongoDB!
✅ How to check if it's working
  1. Open your website on GitHub Pages
  2. Log in as admin and create a task
  3. Go to cloud.mongodb.com → your cluster → Browse Collections
  4. You should see the task saved there! 🎉
⚠️ Free tier note — Render "sleep"
On Render's free plan, your backend goes to sleep after 15 minutes of no traffic. The first request after sleeping takes about 30–60 seconds. This is normal for free hosting. Upgrade to a paid plan ($7/month) to keep it always awake.
